## ADDED Requirements

### Requirement: 应用必须以 macOS 状态栏骨架形态启动
系统 SHALL 提供一个可运行的 macOS 应用入口，并在启动后创建状态栏项，而不是依赖普通窗口应用作为主要入口形态。

#### Scenario: 首次启动时创建状态栏项
- **WHEN** 用户启动应用
- **THEN** 系统必须创建一个可见的状态栏项并使其处于可交互状态

### Requirement: 状态栏应用必须提供占位级基础交互
系统 SHALL 为状态栏项提供基础菜单或等效交互容器，用于承载占位标题、退出入口以及未来功能入口，即使真实价格业务尚未实现。

#### Scenario: 打开状态栏交互时显示占位内容
- **WHEN** 用户点击状态栏项并打开应用交互容器
- **THEN** 系统必须展示占位文案和至少一个退出应用的基础操作

### Requirement: 骨架阶段不得要求真实价格数据
系统 SHALL 在没有真实行情服务和网络请求的情况下正常启动并展示占位状态，确保初始化工程独立可运行。

#### Scenario: 未接入真实 API 时仍可运行
- **WHEN** 应用尚未配置任何真实价格数据来源
- **THEN** 系统必须使用占位显示而不是因为缺少业务实现而启动失败
